# Требования к тестовому заданию

## Технический стек (обязательно)
- ✅ Next.js для роутинга и страниц
- ✅ Tailwind CSS для стилизации (темная тема: brown, white, black)
- ✅ Redux Toolkit для управления состоянием
- ✅ Formik для форм (email валидация)
- ✅ i18next для мультиязычности (AR/EN с RTL для арабского)
- ✅ Strapi CMS для контента

## Компоненты и секции

### 1. HeaderNavigation
- Responsive Navbar с логотипом и ссылками
- Dropdown для "Services" (простой dropdown с brown фоном)
- Иконка поиска, открывающая input
- Поиск редиректит на страницу поиска с результатами (Team и Services)
- Переключатель языка AR/EN с RTL для арабского

### 2. HeroSection
- Фон из изображений/видео из CMS
- Slider с автоплеем для видео, плавные переходы для изображений
- Мультиязычность AR/EN с RTL
- CTA кнопка "Read More" конфигурируемая из CMS (внутренняя или внешняя ссылка)

### 3. Our Team
- Отображение членов команды (изображение, имя, роль)
- Данные из CMS
- Мультиязычность AR/EN с RTL

### 4. Clients
- Отображение логотипов, отзывов или кейсов
- Данные из CMS
- Мультиязычность AR/EN с RTL

### 5. Footer
- Ссылки
- Форма подписки с Formik
- Email валидация
- Защита от дубликатов
- Сохранение в Strapi
- Сообщения об успехе/ошибке
- Мультиязычность AR/EN с RTL

## Страницы

### Homepage
- Все секции: HeaderNavigation, HeroSection, Our Team, Clients, Footer

### Service Detail Page
- Динамический роут `/services/[service-id]`
- Контент из CMS
- Мультиязычность AR/EN с RTL

### Search Page
- Две вкладки: Team и Services
- Пагинация (9-10 элементов на страницу)
- Поиск по запросу из Redux
- Результаты из CMS API
- Мультиязычность AR/EN с RTL

## CMS (Strapi) Коллекции
- Pages (HeroSection контент)
- Services (с slug для динамических страниц)
- Team Members
- Clients
- Subscribers (для email подписки)
- Blog (опционально, если понадобится)

## Важные моменты
- ✅ Вся сайт должен быть из CMS (не статичные данные)
- ✅ Мультиязычность обязательна (i18next)
- ✅ RTL для арабского языка
- ✅ Пагинация из CMS API (не фронтенд)
- ✅ Оптимизация изображений
- ✅ SSG или SSR где необходимо
- ✅ Loading states для API запросов
- ✅ Обработка ошибок
- ✅ Адаптивность
- ❌ Contact Us и About Us страницы НЕ нужны
- ❌ Blog страница НЕ нужна (только ссылка в навигации)
