# Быстрое исправление ошибки 401

## Проблема:
Все запросы к Strapi API возвращают 401 Unauthorized.

## Решение (пошагово):

### 1. Откройте Strapi Admin
http://localhost:1337/admin

### 2. Перейдите в настройки Permissions
- **Settings** (шестеренка внизу слева)
- **Users & Permissions plugin**
- **Roles**
- Откройте роль **Public** (нажмите "Edit")

### 3. Проверьте и отметьте Permissions

В разделе **Permissions** для каждой коллекции:

**Client:**
- ✅ **Read**

**Page:**
- ✅ **Read**

**Service:**
- ✅ **Read**

**Subscriber:**
- ✅ **Read**
- ✅ **Create**

**Team-member:**
- ✅ **Read**

### 4. ВАЖНО - Сохраните!

1. Прокрутите страницу вниз
2. Найдите кнопку **"Save"** (обычно в правом верхнем углу или внизу)
3. Нажмите **Save**
4. **Дождитесь подтверждения** (должно появиться сообщение об успешном сохранении)

### 5. Перезапустите Strapi

После сохранения **обязательно перезапустите Strapi**:

```bash
cd strapi-backend
# Остановите Strapi (Ctrl+C если запущен в терминале)
npm run develop
```

### 6. Проверьте снова

После перезапуска обновите страницу в браузере (http://localhost:3000).

---

## Если все еще не работает:

1. Убедитесь, что вы редактируете роль **Public**, а не другую
2. Проверьте, что галочки действительно отмечены (попробуйте снять и поставить снова)
3. Убедитесь, что нажали **Save** и дождались сохранения
4. Перезапустите Strapi еще раз

---

## Альтернативная проверка через curl:

```bash
curl http://localhost:1337/api/services
```

Должен вернуться JSON (пустой массив, если данных нет), а не ошибка 401.
