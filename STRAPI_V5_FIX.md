# Исправление для Strapi v5

## Проблема:

У вас установлен **Strapi v5.34.0**. В Strapi v5 изменился формат API ответов и возможно способ работы с permissions.

---

## Решение 1: Проверьте формат ответа Strapi v5

В Strapi v5 формат ответа может отличаться. Проверьте напрямую:

```bash
curl http://localhost:1337/api/services
```

Посмотрите, какой формат ответа возвращается.

---

## Решение 2: Обновите трансформацию данных

В Strapi v5 структура ответа может быть другой. Нужно проверить формат и обновить функции трансформации.

---

## Решение 3: Проверьте настройки Permissions в Strapi v5

В Strapi v5 мог измениться интерфейс настройки permissions. Убедитесь, что:
1. Роль Public существует
2. Permissions настроены правильно
3. Сохранены изменения

---

## Решение 4: Используйте API Token

В Strapi v5 API Token может быть более надежным решением:

1. Settings → API Tokens → Create new API Token
2. Token type: **Read-only** или **Full access**
3. Скопируйте токен
4. Добавьте в `.env.local`: `NEXT_PUBLIC_STRAPI_TOKEN=your_token`

---

## Решение 5: Проверьте логи Strapi

Посмотрите логи Strapi в терминале, где он запущен. Там могут быть подсказки о проблеме с permissions или формате запросов.

---

## Быстрая проверка:

Откройте в браузере:
- http://localhost:1337/api/services

Если видите JSON - API работает, проблема в коде.
Если видите 401 - проблема в permissions Strapi.
