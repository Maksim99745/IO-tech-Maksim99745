# Проверка Permissions в Strapi

## Важно: 0 пользователей в Public role - это НОРМАЛЬНО!

Public role предназначена для **анонимного доступа** через API, поэтому там не должно быть пользователей.

---

## Проверка Permissions:

### 1. Откройте роль Public

Settings → Users & Permissions plugin → Roles → Public → Edit

### 2. Проверьте раздел "Permissions"

Должны быть отмечены галочки для каждой коллекции:

**Collection Types:**
- ✅ **Client** → **Read**
- ✅ **Page** → **Read**  
- ✅ **Service** → **Read**
- ✅ **Subscriber** → **Read** + **Create**
- ✅ **Team-member** → **Read**

### 3. Убедитесь, что галочки действительно отмечены

Иногда нужно:
- Кликнуть на галочку дважды (снять и поставить)
- Прокрутить страницу, чтобы увидеть все коллекции
- Проверить, что галочки не серые, а активные

### 4. Сохраните еще раз

- Нажмите **Save**
- Дождитесь сообщения об успешном сохранении

### 5. Перезапустите Strapi

```bash
cd strapi-backend
# Ctrl+C чтобы остановить
npm run develop
```

---

## Проверка через curl:

После перезапуска проверьте:

```bash
curl http://localhost:1337/api/services
```

**Должен вернуться:**
```json
{"data":[],"meta":{"pagination":{...}}}
```

**Если все еще 401:**
- Permissions не сохранены правильно
- Нужно проверить каждую галочку отдельно
- Возможно, нужно создать новую роль Public заново

---

## Альтернатива: Проверьте через браузер

Откройте в браузере:
- http://localhost:1337/api/services
- http://localhost:1337/api/team-members
- http://localhost:1337/api/clients

Если видите JSON (даже пустой) - permissions работают!
Если видите ошибку 401 - permissions не настроены.
